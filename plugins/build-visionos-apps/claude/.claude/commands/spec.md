# /spec

Start a visionOS feature specification. No code until the spec is complete and approved.

## Arguments

$ARGUMENTS - Feature name or brief description of what to build

## Workflow

Load and follow the spec-driven-spatial skill at skills/spec-driven-spatial/SKILL.md.

The spec must answer these questions before any code is written:

1. What is the feature's user-facing job?
2. Which scene model? Window, volume, immersive space, or mixed flow
3. What is the entity lifecycle and ownership model?
4. Are ARKit sessions required? Which providers?
5. Is SharePlay needed? What is shared?
6. Which privacy entitlements are needed?
7. What are the acceptance criteria?

Output a structured spec document. Hand off to /plan when the spec is approved.
